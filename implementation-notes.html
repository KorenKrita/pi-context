user-decided Migrate the complete canonical ACM architecture and behavior from omp-context to pi-context, preferring public Pi interfaces where available. The Pi implementation must preserve observable ACM contracts without mechanically copying OMP-private types or seams.
user-decided Gate Live Agent Sync only on the runtime capabilities it uses; do not inspect or report the Pi host version. If the required AgentSession lifecycle and mutable message-state capabilities are present, synchronization runs. Missing or failed capabilities fall back to the persistent context rebuild and reload recovery path. agent-resolved Associate live AgentSession instances with SessionManager identity through one idempotent, weakly referenced adapter. The adapter probes the getContextUsage() lifecycle seam and each captured session's sessionManager, agent.state, and replaceable messages state. It rebuilds messages from the active SessionManager leaf and never owns tree mutations.
agent-resolved Keep persistent tree mutation in Host Bridge and live-state replacement in a separate compatibility boundary. appendLabelChange(), branchWithSummary(), and buildSessionContext() are public Pi capabilities, but extension tool context exposes only a readonly manager type. Host Bridge performs guarded capability access; the adapter only synchronizes the matching live session after callers schedule it.
agent-resolved Schedule live synchronization only after a structurally verified travel and apply it from the matching tool_execution_end event. The travel tool remains synchronous and truthful about its branch result. Deferring message replacement until the matching tool execution ends preserves the in-flight call/result protocol while preventing failed, indeterminate, or unrelated tool executions from modifying live state.
agent-resolved Preserve a verified traveled branch when live message replacement fails, and surface reload guidance without attempting tree rollback. The persistent SessionManager branch and context-event rebuild remain authoritative. A post-travel live-sync failure clears only the matching pending tool operation, records a terminal diagnostic, and emits a warning; rolling back a valid travel would be more destructive than leaving native accounting stale until reload.
agent-resolved Treat each scheduled synchronization as replaceable per SessionManager, and make the resulting persisted branch the only resume authority. A later travel supersedes stale pending work only for the same manager. Off-path restoration rebuilds from the newly selected leaf, while process resume opens the persisted tree into a new manager/session association instead of retaining any previous-process AgentSession object.
agent-resolved Scope live synchronization strictly to each SessionManager/AgentSession identity pair, including parent/subagent and duplicate extension registrations. The process-wide adapter owns weak session associations and one atomic { toolCallId, preferredLeafId } pending ticket per manager. Duplicate extension runtimes therefore cannot consume each other's tool completion, lifecycle cleanup addresses one manager only, and no working-directory, model, session-file, or global-current-session matching is used.
agent-resolved Use strict Pi TypeBox schemas rather than recreating the OMP runtime-injected Zod seam. Pi exposes TypeBox through @earendil-works/pi-ai. The timeline schema keeps the single-view contract, including omitted view as active mode, without accepting legacy competing booleans.
agent-resolved Make capability observation non-throwing and publish installation state only through a successfully installed wrapper. Frozen/read-only host prototypes and throwing session getters are capability failures, not extension-fatal errors. The wrapper carries the process-wide installation marker, observation never prevents the original lifecycle method from running, and current session capabilities are re-probed when synchronization is scheduled and applied.
agent-resolved Verify message replacement by checking that live state retains the rebuilt message sequence, not the caller's array identity. Live validation against Pi 0.80.6 exposed that the documented Agent.state.messages setter deliberately stores nextMessages.slice(). Reference identity therefore reports a false failure after a successful replacement. Element-by-element sequence identity accepts the host's top-level defensive copy while still detecting thrown or silently ignored replacement when the rebuilt branch differs.
agent-resolved Retire prototype-only synchronization code only after the production capability matrix proves equivalent behavior. The final fixture combines a shrinking travel, a valid in-flight acm_travel call/result pair, provider-context rebuild, archive recovery, and native compaction accounting. Additional fixtures cover unsupported capability, replacement failure, repeated travel, off-path restoration, persistence/resume, lifecycle cleanup, and multi-session/parent-subagent isolation. No prototype runner or duplicated adapter remains.
agent-resolved Keep the independently improved Pi-only /context TUI and route its compaction-aware message reconstruction through Host Bridge. The remote implementation depended on a helper removed by the modular ACM migration. Reusing buildSessionMessages() preserves its calibrated token breakdown and Other accounting while keeping SessionManager reconstruction capability access at the repository's single host seam.
agent-resolved Implement semantic rebase as agent-level use of the existing Pi acm_travel contract, with cold start as the hard safety gate. The underlying Pi tree mutation is identical to local travel. The runtime therefore reports only factual active/projected summary depth and travel deltas; candidate safety remains a semantic judgment performed from earliest to latest, with root treated as a candidate rather than a default.
agent-resolved Keep Pi-native TypeBox schemas, lifecycle seams, /context TUI, and capability-probed adapter while synchronizing the shared ACM guidance and evidence model. The corresponding behavior is adapted to Pi architecture instead of copying OMP-specific package APIs, injected Zod, or host contracts. The extension now always registers its canonical prompt hook because this repository has no integrated-consumer bypass.
user-decided Refine rebase without weakening checkpoint, local-fold, failed-direction, or task-end behavior. The always-on CORE retains every normal transition and a complete local-fold example. Rebase adds a structural-reset criterion alongside cold start rather than replacing the existing fold gate.
user-decided Improve checkpoint invocation predictability by front-loading a task-start preflight and making the checkpoint tool description trigger-first. The prompt follows the user-provided writing-great-skills principles: preflight is the leading word, the first action is explicit, and completion is checkable from the checkpoint tool result.
agent-resolved Keep the always-on CORE below its existing 6,000-character budget while adding the preflight. The duplicate new-chain table row was removed, and vocabulary plus transition wording was tightened without removing any established checkpoint, fold, rebase, failed-direction, or task-end behavior.
agent-resolved Use one advanced reference at a time and reroute when the observable condition changes. This preserves progressive disclosure without trapping archive recovery inside the wrong playbook after a failure. Recovery references now distinguish backup-label rollback from indeterminate branch mutation and use factual refresh/live-sync lifecycle states.
user-decided Send ACM context-usage reminders through Pi's hidden custom-message channel at 30%, 50%, and 70% usage tiers. The reminder must identify itself as ACM-generated and remain invisible in the Pi TUI while participating in model context. The 30% tier is a light opportunity cue, 50% asks the agent to actively look for a safe travel boundary, and 70% is the final and strongest reminder for that context cycle.
user-decided Keep reminders advisory: prefer low active-context usage, but never force summary, fold, rebase, or travel. The agent evaluates whether current task requirements permit a complete and recoverable handoff. Correctness, task continuity, required working state, and recoverability take priority; native compaction remains acceptable for genuinely long tasks.
user-decided Track the highest reminder tier per context cycle; ordinary usage drops do not rearm tiers, while successful ACM travel and native Pi compaction start a new cycle. Within a cycle only a newly reached higher tier produces a reminder, and a single observation that skips tiers emits only the highest one. A new cycle treats its first authoritative post-transition usage as a no-reminder baseline.
agent-resolved Arm reminders from the context lifecycle, deliver at tool_result with steer, and use normal-stop agent_end with followUp as the no-tool fallback. This follows Pi's public Channel 2 transport without modifying tool output. After travel or compaction, immediate getContextUsage() may still reflect pre-transition native accounting, so baseline establishment waits for the next real assistant prompt usage captured at turn_end.
agent-resolved Persist the first authoritative post-transition baseline as an acm:context-usage-state custom entry, then restore reminder-cycle state from baseline entries plus delivered reminder messages. A delivered reminder alone cannot distinguish “baseline not established” from “baseline established below 30%,” and reload must also preserve a baseline already at 30% or 50%. Pi's public appendEntry() channel keeps the baseline marker on the active branch without adding it to LLM context; reconstruction scans only after the latest ACM travel branch summary or native compaction boundary.

Open decisions

None. The capability and recovery verification matrix passes against the repository host fixture. If Pi later exposes an official tool-context interface that atomically navigates the session tree and synchronizes live agent messages, replace and remove the capability-probed AgentSession adapter rather than maintaining both paths.
