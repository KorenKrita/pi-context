Pi Context Implementation Notes

ProvenanceDecisionRationale and constraint
user-decided Migrate the complete canonical ACM architecture and behavior from omp-context to pi-context, preferring public Pi interfaces where available. The Pi implementation must preserve observable ACM contracts without mechanically copying OMP-private types or seams.
agent-resolved Support exactly Pi 0.80.6. The extension, isolated host fixture, and live-session compatibility seam use the same pinned host version. Host-version drift returns an explicit unavailable outcome rather than guessing runtime shapes.
agent-resolved Associate live AgentSession instances with SessionManager identity through one idempotent, weakly referenced adapter. Pi exposes AgentSession.agent and a public mutable agent.state.messages contract. The adapter captures the association at the pinned getContextUsage() lifecycle point, rebuilds messages from the active SessionManager leaf, and assigns the public message state. It never owns tree mutations.
agent-resolved Keep persistent tree mutation in Host Bridge and live-state replacement in a separate compatibility boundary. appendLabelChange(), branchWithSummary(), and buildSessionContext() are public Pi capabilities, but extension tool context exposes only a readonly manager type. Host Bridge performs guarded capability access; the adapter only synchronizes the matching live session after callers schedule it.
agent-resolved Schedule live synchronization only after a structurally verified travel and apply it from the matching tool_execution_end event. The travel tool remains synchronous and truthful about its branch result. Deferring message replacement until the matching tool execution ends preserves the in-flight call/result protocol while preventing failed, indeterminate, or unrelated tool executions from modifying live state.
agent-resolved Preserve a verified traveled branch when live message replacement fails, and surface reload guidance without attempting tree rollback. The persistent SessionManager branch and context-event rebuild remain authoritative. A post-travel live-sync failure clears only the matching pending tool operation, records a terminal diagnostic, and emits a warning; rolling back a valid travel would be more destructive than leaving native accounting stale until reload.
agent-resolved Treat each scheduled synchronization as replaceable per SessionManager, and make the resulting persisted branch the only resume authority. A later travel supersedes stale pending work only for the same manager. Off-path restoration rebuilds from the newly selected leaf, while process resume opens the persisted tree into a new manager/session association instead of retaining any previous-process AgentSession object.
agent-resolved Scope live synchronization strictly to each SessionManager/AgentSession identity pair, including parent/subagent and duplicate extension registrations. The process-wide installation marker owns only WeakMap indexes and WeakRef session values. Each extension runtime keeps its own pending tool-call correlation, lifecycle cleanup addresses one manager only, and no working-directory, model, session-file, or global-current-session matching is used.
agent-resolved Use strict Pi TypeBox schemas rather than recreating the OMP runtime-injected Zod seam. Pi exposes TypeBox through @earendil-works/pi-ai. The timeline schema keeps the single-view contract, including omitted view as active mode, without accepting legacy competing booleans.
agent-resolved Discover the pinned host manifest by walking ancestor node_modules directories. Bun's bundled fixture does not reliably resolve the scoped host package through createRequire(). Filesystem discovery is deterministic in both source-first and isolated fixture layouts and still requires the exact 0.80.6 manifest.

Open decisions

None for successful travel, synchronization failure recovery, repeated travel, off-path restoration, persistence/resume, lifecycle cleanup, or multi-session/subagent isolation. Final combined verification and prototype retirement remain implementation work rather than open design decisions.
