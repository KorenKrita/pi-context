agent-resolved Require acm_travel to execute alone in its assistant tool batch, force sequential dispatch, and reject a detected mixed batch before any session mutation. A successful travel replaces the live/session branch at that event, so sibling results from the removed assistant message can become orphaned and be sanitized away even though their external side effects already occurred. Sequential dispatch makes ordering deterministic; the runtime guard remains the safety boundary for both call orders, while canonical guidance prevents avoidable retries.

Without normalization, JSON.stringify() preserves CRLF inside generated runtime strings and the strict generated-guidance check reports a false stale artifact before tests run. The generator now normalizes source and comparison input, while the repository attribute keeps normal working trees deterministic across operating systems.
user-decided Replace the mandatory ACM preflight, transition table, and gate checklists with compression-as-intelligence doctrine, recognition cues, and default agent autonomy. agent-resolved Treat manual /tree navigation as a first-class context transition: reset volatile ACM runtime state on session_tree, shape plain summarize requests through session_before_tree into the seven-slot handoff, and count every active-branch summary or compaction as a reminder-cycle boundary on restore. agent-resolved Make every prompt surface name its concrete next move: the checkpoint cue teaches the checkpoint→travel chain, each reminder tier carries one tool step, and the handoff contract states line-per-slot shape with an immediately executable NEXT. Weaker models act on named tools, not abstract advice: "weigh a fold" gives no callable action, while "acm_travel targeting this point folds that process" does. The additions stay advisory — no tier or cue mandates a call — so stronger models keep full judgment while the action gradient still points the same direction at every capability level.
user-decided Remove fixed prompt, CORE, tool-description, Skill, character, and token ceilings as product constraints; optimize only for observable ACM and task effect. This supersedes earlier 6,000/6,500-character and similar anti-sprawl guards. Except for real host or protocol limits, length is diagnostic rather than normative. Content is retained, removed, inlined, or progressively disclosed according to behavioral effect, information hierarchy, and maintainability—not to satisfy an arbitrary budget.

Open decisions

  • open: Validate and adopt an ACM judgment contract that optimizes task and attention effect rather than tool frequency, context reduction, or fixed prompt-length budgets.
  • open: Decide how agent-facing handoff input becomes mechanically easy while the future agent receives clear, trustworthy, directly usable information.
  • open: Establish continuation-safe target and post-travel authority semantics so the future agent can trust the handoff without a verification ritual.
  • open: Make advanced guidance discoverable without requiring runtime to infer semantic complexity or reintroducing mandatory preflight.
  • open: Replace activation-oriented evaluation with task-, continuity-, attention-, and transition-harm evidence, including cases whose original real-world task is not exactly reproducible.

If Pi later exposes an official tool-context interface that atomically navigates the session tree and synchronizes live agent messages, replace and remove the capability-probed AgentSession adapter rather than maintaining both paths.
